```txt
Live invoice preflight: BLOCKED - No channels found
Live route probe:       BLOCKED - No tested fee or MPP setting produced a dry-run route.
```

That is expected: pending, unfunded channels are not usable payment channels, so the product should block the route rather than claim payability.

## Dashboard Test

The dashboard can reproduce this proof from the browser:

```txt
Source: Live RPC
RPC URL: http://127.0.0.1:8227
API URL: http://127.0.0.1:8787
Action: Run live test
```

Without an invoice, the dashboard shows live RPC status, ready-channel inventory, and historical pending or failed channel opens. With a fresh Node C invoice pasted into the invoice field, it runs invoice preflight and route probes against the same live channel state. The expected judge-facing result is a ready RPC, one ready enabled channel, a cautious preflight verdict because the channel is private and older failures remain in history, and a payable route probe with estimated fee `0`.
